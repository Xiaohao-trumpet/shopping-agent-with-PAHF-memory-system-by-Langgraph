# 商品与店铺评价分析（发展潜力）模块

> 在原「服务体验评价（CSAT）」之外，新增**商品维度**的评价采集与分析：可量化、可解释地
> 评估**单个商品**和**整个店铺**的「发展潜力」，并用 AI 生成评价、生成经营洞察，配套一套
> 美化后的分析后台。

## 为什么要做

原系统只有会话级 CSAT（评估「这次客服服务好不好」），无法回答运营真正关心的问题：
**哪些商品在涨、哪些在衰、店铺整体健康度如何、下一步该加投还是止损。**
本模块把「评价」从服务体验扩展到商品本身，形成可分析的数据资产。

## 架构（`backend/analytics/`）

| 文件 | 职责 |
| --- | --- |
| `review_store.py` | SQLite 商品评价存储；聚合统计（评分分布、情绪占比、标签/维度词频、**动量**、趋势分桶）；确定性 Demo 评价灌注 |
| `potential.py` | **确定性、可解释**的潜力评分引擎（商品 + 店铺），零依赖 LLM |
| `ai_reviewer.py` | AI 层：生成评价 / 分析商品 / 店铺经营洞察；**离线自动降级**为规则引擎 |
| `service.py` | 门面：拼装商品元数据 + 销量 + 评分 + 潜力 + AI；带缓存；对外提供接口 |

设计沿用项目一贯的轻量约定：单 SQLite 文件、`sqlite3.Row`、无 ORM、单机一键跑通。

## 潜力评分（可解释）

潜力分 = 五个归一化子项的加权和 × 100，权重刻意把**动量（趋势）**放在最高——潜力关乎趋势而非快照：

| 子项 | 权重 | 含义 |
| --- | --- | --- |
| 满意度 satisfaction | 0.28 | 当前均分水平 `(avg-3)/1.5` |
| 增长势能 momentum | 0.30 | 评分趋势（近30天 vs 更早）+ 评价加速度 |
| 口碑情绪 sentiment | 0.17 | 好评占比 − 差评占比×1.2 |
| 评价体量 volume | 0.15 | 评价数量（对数缩放） |
| 销售拉动 demand | 0.10 | 已付款订单的销量（对数缩放） |

分档：`≥72 明星`、`58–71 潜力`、`42–57 平稳`、`<42 预警`、`无评价 待评价`。
每个商品都会返回 `drivers`（各子项贡献、正负基调、一句话理由），后台据此解释「为什么是这个分」。

**店铺潜力** = 各商品潜力按评价量加权聚合，并给出分档计数、分类潜力排行、明星榜与预警榜。

## AI 层（在线用模型，离线用规则）

- **生成评价** `generate_reviews`：为商品合成真实口吻的中文评价（老师要求的「让 AI 生成评价」），
  可用于稀疏商品补足或 Demo 预热。
- **商品洞察** `analyze_product`：把评价+统计凝练为 优点/缺点/维度主题/潜力叙述/建议动作/风险等级。
- **店铺洞察** `summarize_store`：面向运营的经营摘要（亮点/风险/机会/行动建议）。

三者均带确定性降级：无 `API_KEY` 或调用失败时自动走规则引擎，返回体用 `generated_by: "ai" | "heuristic"`
标识，前端据此打「AI 生成 / 规则引擎」徽标。**因此离线也能完整演示。**

## 接口

### 顾客端（商城，无需鉴权）
```
GET  /api/v1/shop/review-tags
GET  /api/v1/shop/products/{product_id}/reviews?limit=&sentiment=
POST /api/v1/shop/products/{product_id}/reviews   {customer_id, rating, title, content, tags}
```

### 后台端（需管理员 Bearer Token）
```
GET  /api/v1/admin/analytics/store?ai=0|1                     # 店铺潜力 + 经营洞察
GET  /api/v1/admin/analytics/products?sort=score|rating|reviews|trend&order=&category=&limit=
GET  /api/v1/admin/analytics/products/{product_id}?ai=0|1     # 深度分析：统计/评价/潜力驱动/洞察
POST /api/v1/admin/analytics/products/{product_id}/ai-insight # 强制生成 AI 洞察
POST /api/v1/admin/analytics/products/{product_id}/generate-reviews  {n, skew, persist}
```

## 前端

- **商城商品详情**：评价区（均分/评分分布/好评率/近期趋势、标签、好评/差评筛选、写评价表单）。
- **后台「潜力分析」标签页**：
  - 店铺潜力仪表盘（潜力分环形图 + 分档 + KPI + 分档计数）；
  - AI 经营洞察卡（亮点/风险/机会/行动，「🤖 生成 AI 洞察」按钮）；
  - 分类潜力条形榜；
  - **商品发展潜力榜**（可按潜力分/评分/评价数/势能排序，分值条 + 分档徽标 + 趋势箭头）；
  - 点击进入**商品深度分析抽屉**：潜力驱动因子拆解、评分分布、好评/差评、销量、
    标签、评价明细，以及「生成 AI 洞察」「AI 生成评价」按钮。

## 配置项
```
REVIEW_DB_PATH=./data/reviews/reviews.db
REVIEW_AUTO_SEED=true      # 首启按商品目录确定性灌注一批评价（含涨/跌/两极分化画像）
```

## 试用路径
1. `python run_all.py`（首启自动灌 500+ 条示例评价，画像覆盖 明星/潜力/平稳/预警）。
2. **商城** → 打开任意商品详情 → 看评价区、写一条评价。
3. **后台管理 → 潜力分析** → 看店铺潜力与洞察 → 点榜单里某个「预警」商品进抽屉 →
   看驱动因子与差评 → 点「AI 生成评价」补几条好评 → 潜力分随近期动量回升。
